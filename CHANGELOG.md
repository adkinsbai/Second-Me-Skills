# Second-Me-Skills 发布日志

> 版本号说明：本项目以 `V0.x` 表示“产品迭代代次”。每次迭代聚焦一组体验/审美改进。

## V0.1（当前基线）
- 首页：在未 `/?updateIntro=1` 时改为“匹配开启引擎”入口（旋转地球 + 网络连线）
- 匹配开启动画：模拟匹配成功/失败提示并进入 `/matches`
- `/matches`：顶部标题改为“用户名 + 分身”
- UI：开始界面与“心动预选”加入天空白云氛围装饰
- 登录态头部：修复 `AppHeader` 默认标题导致“两个丘比”
- 数字体/主人信息库：加入 `.secondme/owner_information.json`，并让“我的 Agent 介绍”基于主人信息库生成

## V0.2（已完成）
- 匹配引擎后端：`/api/matches/seed` 根据用户 `心动阈值` 随机决定是否达标，并正确设置 `match.status` 与 `matchScore.totalScore`
- 匹配开启界面：成功阶段文案调整为分阶段展示“正在解锁 / 目前的已解锁”
- 失败阶段文案保持“您的心动对象似乎还在游荡…”

## V0.3（进行中/下一步）
- 匹配开启页：浪漫风「流动心形背景 + 中央炫彩 Q（丘比）」纯 CSS/SVG（已移除 WebGL 地球，减少解锁后跳转异常）
- 多维度报告：演示 `reportJson` 扩展为「综述 / 细分观察 / 对话证据 / 风险 / 下一步」；报告页展示「丘比总览」与各维度卡片
- `/matches` 增强：增加“解锁原因/状态提示条”（更温馨、更明确）
- 补齐加载与空状态：Skeleton/错误提示更一致，避免“点了没反应”
- 优化动效：对低性能设备降低动画强度
- 首页：`useSearchParams` 外包 `Suspense`，修复 `next build` 预渲染报错

## V0.4
- 主人信息库体验：首页增加“主人信息库概览/清空按钮”，便于重开体验
- “我的 Agent 介绍”刷新：增加刷新按钮与明显的更新反馈

## V0.5
- 真实感反馈：匹配成功后延迟展示“Agent 正在聊天中…”再进入 `/matches`
- `/matches/[id]`：Agent 对话与评分报告之间的引导文案更自然

## V0.6
- 视觉统一：建立更一致的卡片圆角、阴影、按钮层级与间距
- 天空/云朵：统一到全局设计（避免页面风格漂移）

## V0.7
- 可用性：增加“重试匹配”“返回心动设置”的更清晰可达入口
- 表单/按钮交互：禁用态更明确，防止重复点击造成多次 seed

## V0.8
- 无障碍：完善焦点态、ARIA、键盘可用性
- 动效友好：支持 `prefers-reduced-motion` 时减少旋转与闪烁

## V0.9
- 性能：减少不必要 rerender 与请求（尤其是 matches 列表、agent 介绍摘要）
- 稳定性：补齐超时/失败回退策略，避免卡死

## V0.10（明天目标）
- 打磨到“可演示/可上线”的完整体验：视觉、交互、文案、稳定性全部收敛
- 输出最终版本的体验清单与下一步“上线 SecondMe”的准备项
