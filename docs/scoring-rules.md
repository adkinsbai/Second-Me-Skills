## Agent 心动值打分规则（v1）

本规则仅作用于 **Agent 对 Agent 对话页底部的「心动值」进度条**，不影响数据库中的 `MatchScore` 原始报告分数。

### 1. 基本设定

- 心动值取值范围：0～100。
- 新增演示对象进入 Agent 对话页时的默认心动值：**60 分**。
- 每当「对方 Agent」（`agent_target`）说完一句话后，根据该句内容对心动值进行 **-1、0、+1** 的微调。
- 当心动值：
  - **＜ 55**：认为本轮对话氛围偏低，自动停止实时演示（`stopReason = "low"`），两侧 Agent 不再继续自动聊天。
  - **≥ 用户心动阈值**：标记为「重点关注对象」，自动停止实时演示（`stopReason = "high"`），并在心动值 ≥ 80 时触发爱心动画。

### 2. 文本特征与加减分

以下规则在代码中通过正则匹配实现（详见 `src/app/matches/[id]/agent-chat/page.tsx` 中的 `computeHeartDelta` 函数），每次 **最多加 1 分或减 1 分**：

#### 2.1 明显正向信号（+1 分）

当一句话中 **至少命中一条正向模式**，且 **没有命中明显负向模式** 时，心动值 +1。

正向模式示例：

- 相处/陪伴意愿：
  - 包含「一起」「陪伴」「陪你」等词。
- 情绪理解与在乎：
  - 包含「理解」「共鸣」「在乎」「照顾」等词。
- 开放与尝试：
  - 包含「愿意」「可以试试」「可以一起」等词。
- 边界与安全感：
  - 包含「边界」「尊重」「安全感」等词。
- 共同活动偏好：
  - 包含「看展」「旅行」「出门玩」「一起玩」等词。

#### 2.2 明显负向信号（-1 分）

当一句话中 **至少命中一条负向模式**，且 **没有命中正向模式** 时，心动值 -1。

负向模式示例：

- 表达明确的拒绝或没兴趣：
  - 「不太喜欢」「没兴趣」「算了」
  - 「不合适」「可能不太适合」「不太想」

#### 2.3 轻度情绪表达（+1 分）

当一句话没有命中正向/负向模式，但包含情绪词时，会轻微加 1 分，认为是「在认真分享自己的状态」：

- 匹配关键词：「压力」「累」「焦虑」「紧张」等。

#### 2.4 其余情况（0 分）

- 不命中以上任何模式时，不调整心动值（+0）。

### 3. 阈值与停止条件

- **下限停止线：55 分**
  - 一旦心动值跌破 55 且当前还没有设置 `stopReason`：
    - 设置 `stopReason = "low"`；
    - 关闭自动演示开关（`autoDemoRef.current = false`），停止继续调用 `/agent-chat/ai-turn`。
- **上限触发线：用户心动阈值（来自「心动设置」页）**
  - 当心动值 ≥ `heartThreshold` 且当前还没有 `stopReason`：
    - 设置 `stopReason = "high"`；
    - 关闭自动演示；
    - 若此时心动值 ≥ 80，则额外触发爱心动画（`showLove = true`），作为「高匹配成功」的视觉反馈。

### 4. 实现位置

- 规则函数：`computeHeartDelta`  
  路径：`src/app/matches/[id]/agent-chat/page.tsx`
- 应用时机：
  - 仅在收到 **对方 Agent**（`senderType === "agent_target"`）的新消息时调用。
  - 先用 `computeHeartDelta` 计算本轮增减，再根据结果与阈值更新心动值与停止状态。
