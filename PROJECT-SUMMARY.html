应用是做什么的

丘比是一款面向「真实心动匹配」的交友应用：用户设置心动偏好与阈值，系统在候选池中做双向条件筛选，再结合资料向量与合拍模型打分，生成匹配结果；解锁后可进行真人聊天，并查看多维度评分报告。产品同时集成 SecondMe 数字体能力（OAuth、流式聊天、用户画像等），并包含丘比小镇社区发帖与双人对话等模块。

黑客松 / 演示向 匹配引擎 + 可视化流水线 PostgreSQL + Prisma

已实现的核心功能（概览）

  • 账号与登录：SecondMe OAuth2、本地邮箱密码（bcrypt）、访客、忘记/重置密码。
  • 资料与心动设置：昵称/头像/简介/性别/年龄、期望对象、地区、匹配类型、聊天节奏、心动阈值等（UserPreference）。
  • 匹配流水线：多阶段筛选（池扫描 → 有效账号 → 双向心动条件 → 向量/合拍 → 双向阈值），/api/matches/seed 返回阶段统计；首页/匹配开始页可展示进度与动画。
  • 匹配列表与详情：历史匹配、综合分、入口至聊天/报告/关系页；解锁后与对方对称的匹配记录。
  • 真人聊天：REST 收发消息、SSE 推送新消息、乐观发送、已读（MatchRead）、分页加载更早消息、图片 DataURL、微信风格气泡 UI。
  • 评分报告 / 关系洞察：多维度分数与说明文案（与 MatchScore、demo 报告数据结合）。
  • 丘比小镇：发帖、探索、我的需求、会话列表、小镇内双人聊天与已读。
  • 数字体相关（SecondMe）：首页/Owner 侧流式聊天、Act 流式接口等；双方 Agent 初识对话相关代码保留，产品入口可按版本关闭或隐藏。
  • 其他页面：隐私政策、自我介绍测验、Onboarding 变体等。

技术栈

层级 技术选型
框架 Next.js 14（App Router，SSR + Route Handlers）
语言 / UI TypeScript 5 · React 18 · Tailwind CSS 3 · 全局 CSS 动画
数据 PostgreSQL（如 Neon）· Prisma 5 ORM · schema.prisma 管理模型
实时 聊天：SSE（/api/matches/[id]/chat/stream）轮询合并新消息；客户端乐观更新
外部服务 SecondMe API（OAuth、/api/secondme/chat/stream、用户 shades 等）
匹配算法 自研 matchPipeline + userModeling / scoreCompatibility；资料向量存 UserEmbedding（可演进为 pgvector + 真实 Embedding）
部署 典型为 Vercel Serverless（需配置 DATABASE_URL 与 SecondMe 环境变量）

页面路由一览

以下为应用中主要 page.tsx 对应路由及作用摘要（以仓库当前结构为准）。

路由 功能概览
/ 首页：登录态下展示匹配网络/入口、与 SecondMe 相关主流程入口等。
/auth 登录与注册：SecondMe OAuth、本地账号表单等。
/auth/forgot-password 忘记密码：提交邮箱申请重置链接。
/auth/reset-password 重置密码：带 token 的表单页。
/onboarding 新用户引导：分步填写基础信息与偏好。
/onboarding/paradise 引导子流程（活动/乐园向页面）。
/intro/quiz 自我介绍测验：问卷式补充画像。
/settings/heartbeat 心动设置：阈值、期望性别/年龄/地区、匹配类型、节奏等。
/match-start 开启匹配：调用 seed API，展示匹配流水线阶段与动画。
/matches 匹配列表：历史对象、分数与未读等。
/matches/[id] 匹配详情：对方资料摘要、多维评分、进入聊天/报告/关系等入口。
/matches/[id]/chat 真人聊天：SSE + 消息列表、输入区、已读、图片等。
/matches/[id]/report 评分报告：综合分与维度说明。
/matches/[id]/relationship 关系洞察：基于匹配的延伸分析展示。
/matches/[id]/agent-chat 双方 Agent 初识对话（代码保留；产品策略上可隐藏入口）。
/users/[id] 用户公开主页：查看他人资料与标签。
/town 丘比小镇首页：我的帖子/对话入口与帖子流。
/town/explore 探索：按分类浏览需求帖子。
/town/messages 小镇消息列表：会话入口。
/town/my-needs 我发布的需求管理。
/town/conversations/[id] 小镇双人对话：帖子上文上下文 + 聊天。
/privacy 隐私政策静态说明页。

各页面说明（卡片）

首页

/

应用主入口：展示品牌氛围与核心操作（如匹配、SecondMe 相关能力）；未登录引导至认证。

认证

/auth · /auth/forgot-password · /auth/reset-password

统一处理第三方 OAuth 与本地账号；支持密码找回与重置流程。

引导与测验

/onboarding · /onboarding/paradise · /intro/quiz

降低冷启动成本：分步收集信息；测验补充画像用于匹配与展示。

心动设置

/settings/heartbeat

配置「谁会进你的池子」：阈值、地区、年龄、类型、节奏等，驱动 seed 与流水线。

开启匹配

/match-start

触发 POST /api/matches/seed，展示各阶段人数与视觉反馈，成功则跳转匹配详情。

匹配列表 / 详情

/matches · /matches/[id]

浏览历史匹配；详情页展示分数、操作入口（聊天、报告、洞察）。

真人聊天

/matches/[id]/chat

双人对聊：HTTP 发送、SSE 收消息、已读同步、历史分页、图片与输入体验优化。

报告与关系

/matches/[id]/report · /matches/[id]/relationship

结构化呈现匹配解释与关系向洞察，辅助用户决策是否深聊。

Agent 双聊（可选）

/matches/[id]/agent-chat

基于 SecondMe 流式接口的双方 Agent 对话；业务上可与「真实匹配」策略解耦。

用户主页

/users/[id]

查看他人公开资料与标签，可跳转发起小镇或匹配相关动作（视业务逻辑）。

丘比小镇

/town · /town/explore · /town/messages · /town/my-needs

社区发帖、分类探索、会话列表与个人需求管理，与「心动匹配」数据模型分离。

小镇对话

/town/conversations/[id]

围绕帖子上下文的 1v1 聊天，独立消息表与已读模型。

隐私

/privacy

合规与信任：数据使用与隐私说明。
