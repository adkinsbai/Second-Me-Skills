Design handoff

丘比（Qiubi）— UI/设计沟通稿

给 UI/视觉设计师的单页静态稿：产品摘要 + 主要界面参考图。无需打开 App、不嵌入实时试玩页。

要「单文件、图片全在 HTML 里」发给对方？在已有截图的前提下执行 npm run bundle:designer，会生成 public/designer-handoff-embedded.html：PNG 与 town-schematics.html 均以 data: 内嵌，只发这 1 个 HTML 即可（文件会较大，属正常）。
更新内嵌图：npm run capture:ui → 再 npm run bundle:designer。
多文件打包法（不跑 bundle 时）：
1）本文件 designer-handoff.html 与 ui-screenshots/、town-schematics.html 放同一层目录后 zip；
2）或浏览器打印 → 另存为 PDF；
3）更新截图先 npm run dev:3456，再 npm run capture:ui。

一、产品摘要看这个就够

丘比是「真实心动匹配」向交友应用：用户配置心动偏好与阈值，系统做候选池 双向条件筛选，结合 资料向量与合拍分 产出结果；匹配解锁后可 真人聊天 与查看 评分报告/关系向洞察。产品还集成 SecondMe 数字体与 丘比小镇（社区发帖、探索、消息、1v1 对话）。

主流程 匹配流水线可视化 小镇 / 对话 移动端优先

设计师需要知道的能力边界（一句话）

  • 账号：SecondMe OAuth、本地邮箱、访客、找回密码。
  • 核心：引导 / 心动设置 / 开启匹配 / 匹配列表与详情 / 聊天（类微信气泡+SSE）/ 报告与关系页。
  • 社区：/town 系列（首页、探索、消息、我的需求、帖子下对话等）。

更细的界面与组件约定见仓库 UI-SPEC.md、UI-SPEC-FIGMA.md、FEATURES.md；本页负责一眼看全主要屏 + 可分享。
